# Pitch Patterns

Frameworks for creating compelling investor pitch materials.

## When to Use This Skill

- Running the `/pitch` command
- Preparing for investor meetings
- Writing fundraising materials
- Practicing pitch delivery

## Pitch Deck Frameworks

### Sequoia Capital Framework (Series A+)

| Slide | Purpose | Time |
|-------|---------|------|
| 1. Company purpose | One-line description | 15s |
| 2. Problem | The pain you're solving | 60s |
| 3. Solution | How you solve it | 60s |
| 4. Why now | Market timing | 30s |
| 5. Market size | TAM/SAM/SOM | 45s |
| 6. Product | Demo or screenshots | 90s |
| 7. Business model | How you make money | 45s |
| 8. Competition | Your positioning | 45s |
| 9. Team | Why you'll win | 30s |
| 10. Financials | Projections and metrics | 60s |
| 11. The ask | What you need | 30s |

### YC Framework (Pre-seed / Seed)

| Slide | Purpose | Time |
|-------|---------|------|
| 1. What you do | One sentence | 10s |
| 2. Problem | Show the pain | 30s |
| 3. Solution | Show the product | 30s |
| 4. Traction | Prove it works | 30s |
| 5. Market | Show it's big | 15s |
| 6. Team | Show you can do it | 15s |
| 7. Ask | What you need | 10s |

**YC rule: 7 slides, 2 minutes.**

## Storytelling Patterns

### The Hero's Journey (Product)
```
1. Ordinary world (user has a problem)
2. Call to adventure (discovers your product)
3. Tests and challenges (onboarding, learning)
4. Transformation (the "aha" moment)
5. Return with the elixir (user's life is better)
```

### PAS Framework (Problem-Agitation-Solution)
```
1. Problem: State the problem clearly
2. Agitation: Make the pain worse -- quantify the cost
3. Solution: Present your product as the relief
```

### Before/After/Bridge
```
1. Before: Here's the world without us (painful)
2. After: Here's the world with us (beautiful)
3. Bridge: Here's how we get there (your product)
```

## Market Sizing Methods

### Top-Down
```
Total industry revenue (from reports)
  x Percentage in your segment
  x Percentage in your geography
  = TAM
```
**Weakness:** Relies on analyst reports, often inflated.

### Bottom-Up (Preferred by investors)
```
Number of potential customers
  x What they'd pay per year
  = TAM

Number you can reach with current channels
  x What they'd pay
  = SAM

Realistic capture rate (1-5% for seed, 5-15% for Series A)
  x SAM
  = SOM
```
**Strength:** Based on observable, countable customers.

## Investor Objection Patterns

| Objection | What They Mean | How to Respond |
|-----------|---------------|---------------|
| "It's too early" | Not enough traction | Show validation signals, explain milestones |
| "The market is small" | TAM isn't compelling | Reframe the market, show expansion potential |
| "What about [BigCo]?" | Competitive risk | Show your wedge, speed advantage, focus |
| "Your CAC seems high" | Unit economics concern | Show path to decreasing CAC with scale |
| "How do you defend this?" | Moat question | Network effects, data, switching costs, speed |
| "The team is incomplete" | Missing key roles | Show awareness and hiring plan |
| "What if you can't raise more?" | Sustainability | Show path to break-even or profitable small |

## Stage-Appropriate Metrics

### Pre-Seed
| Show | Don't Worry About |
|------|-------------------|
| Waitlist signups | Revenue |
| Letters of intent | Unit economics |
| User interviews (N=20+) | Cohort analysis |
| Prototype feedback | Detailed financials |
| Founder-market fit | CAC/LTV |

### Seed
| Show | Don't Worry About |
|------|-------------------|
| Paying customers | Profitability |
| MRR and growth rate | Detailed CAC |
| Retention/engagement | Multi-year projections |
| NPS or user satisfaction | Enterprise sales metrics |
| Unit economics directionally | Precision in TAM |

### Series A
| Must Show | Red Flag If Missing |
|-----------|-------------------|
| MRR > $50K and growing | No clear path to $1M ARR |
| Month-over-month growth > 15% | Flat or declining growth |
| LTV:CAC > 3:1 | Unclear unit economics |
| Net revenue retention > 100% | High churn with no fix |
| Clear go-to-market playbook | "We'll figure it out" |
