# Ship Page Patterns

Frameworks for creating high-converting landing pages.

## When to Use This Skill

- Running the `/landing` command
- Writing landing page copy
- Optimizing conversion rates
- Creating marketing content from product features

## Landing Page Structure

```
┌─────────────────────────────────┐
│         HERO SECTION            │  ← Above the fold
│  Headline + Subheadline + CTA   │     (most important)
├─────────────────────────────────┤
│       SOCIAL PROOF BAR          │  ← Logos, metrics
├─────────────────────────────────┤
│     FEATURE SECTION (3-6)       │  ← Benefits, not features
├─────────────────────────────────┤
│       HOW IT WORKS              │  ← 3 simple steps
├─────────────────────────────────┤
│      TESTIMONIALS               │  ← Real customer quotes
├─────────────────────────────────┤
│        PRICING                  │  ← Clear tiers
├─────────────────────────────────┤
│          FAQ                    │  ← Objection handling
├─────────────────────────────────┤
│       FINAL CTA                 │  ← Last chance
└─────────────────────────────────┘
```

## Copywriting Frameworks

### PAS (Problem-Agitation-Solution)
```
Problem:    "Tired of losing customers to slow response times?"
Agitation:  "Every minute of delay costs you $X in lost revenue.
             Your competitors are already faster."
Solution:   "[Product] responds in under 100ms, keeping your
             customers engaged and your revenue growing."
```

### AIDA (Attention-Interest-Desire-Action)
```
Attention:  Bold headline that stops scrolling
Interest:   "Here's why X matters more than you think"
Desire:     "Imagine if you could [outcome] in [time]"
Action:     "Start free today"
```

### BAB (Before-After-Bridge)
```
Before:  "Right now, you're spending 10 hours/week on X"
After:   "What if X took 10 minutes instead?"
Bridge:  "[Product] automates the entire process"
```

## Above-the-Fold Rules

| Element | Required | Best Practice |
|---------|----------|--------------|
| Headline | Yes | 5-10 words, benefit-focused |
| Subheadline | Yes | 1-2 sentences, explains how |
| CTA button | Yes | High contrast, action verb |
| Supporting text | Recommended | "Free trial" / "No credit card" |
| Visual | Recommended | Product screenshot or hero image |
| Navigation | Optional | Minimal, don't distract from CTA |

**Test:** Cover everything below the fold. Does the hero alone tell the story?

## CTA Best Practices

### Button Copy

| Good | Bad | Why |
|------|-----|-----|
| Start Free Trial | Submit | Action-oriented |
| Get Started Now | Sign Up | Lower friction |
| See It in Action | Learn More | Specific outcome |
| Try It Free | Buy Now | Removes risk |

### CTA Placement
- **Hero:** Primary CTA (most important)
- **After features:** Secondary CTA (for engaged scrollers)
- **After testimonials:** Reinforcement CTA (social proof + action)
- **Bottom:** Final CTA (last chance with urgency)

### Conversion Boosters Below CTA

| Booster | Effect |
|---------|--------|
| "No credit card required" | +14% conversion |
| "Free for up to X users" | Removes cost objection |
| "Setup in 2 minutes" | Removes effort objection |
| "Join X,000+ teams" | Social proof |

## Feature Section Patterns

### Icon + Headline + Description (Most common)
```
[Icon]
Save 5 Hours Every Week
Automate repetitive tasks with one-click workflows
that run while you focus on what matters.
```

### Alternating Image + Text
```
Left: [Screenshot]    Right: [Text + benefit]
Right: [Text + benefit]    Left: [Screenshot]
```

### Comparison Table (vs. competitors or manual way)
```
| | Manual Way | With [Product] |
|--|-----------|---------------|
| Time | 5 hours | 10 minutes |
| Errors | Frequent | Zero |
| Cost | $500/mo | $29/mo |
```

## FAQ as Objection Handling

The FAQ section is not for user documentation. It's for handling buying objections:

| Objection | FAQ Question |
|-----------|-------------|
| "Is it secure?" | "How do you protect my data?" |
| "Is it expensive?" | "What does it cost?" |
| "Will it work for me?" | "Who is [Product] for?" |
| "Can I leave?" | "Can I export my data?" |
| "Is it reliable?" | "What's your uptime guarantee?" |
| "Do I need technical skills?" | "How easy is it to set up?" |

## Tone Guidelines

| Tone | Characteristics | Example |
|------|----------------|---------|
| Professional | Clean, authoritative, precise | "Enterprise-grade security for growing teams" |
| Casual | Friendly, conversational, warm | "Finally, project management that doesn't suck" |
| Bold | Confident, provocative, energetic | "Kill your spreadsheets. Build with [Product]." |
